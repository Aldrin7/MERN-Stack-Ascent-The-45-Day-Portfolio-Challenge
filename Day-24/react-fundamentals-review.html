Component Fundamentals

JSX, Props, and Component Composition

📝 JSX Syntax Patterns
// Component with JSX function ProductCard({ title, price }) { return (

{title}

Price: ${price}

); } // Usage

🔧 Props System

Props pass data from parent to child components. They're read-only in the child.

Examples: title, onClick, children, styles

📦 Component Composition

Build complex UIs by combining simple components. Use children props for flexible layouts.

Examples: Container → Card → Button pattern

🎯 JSX Expressions

Embed JavaScript expressions in JSX using curly braces. Perfect for computed values.

Examples: {1 + 2}, {item.name}, {isActive ? 'Yes' : 'No'}

🧩 Component Patterns

Presentational components focus on UI, container components handle logic.

Examples: Card (UI) vs ProductList (logic)
🔄

State Management

useState Hook and Data Flow

📊 State Patterns
// Basic State const [count, setCount] = useState(0); // Object State const [user, setUser] = useState({ name: '', email: '', age: 0 }); // Array State const [items, setItems] = useState([]); function addItem(newItem) { // ✅ Don't mutate state directly setItems([...items, newItem]); }

📝 Immutable Updates

Never mutate state directly. Always create new state values for React to detect changes.

Examples: [...array], {...object}, array.filter()

🔄 State Lifting

Move state up to a common parent when multiple components need to share data.

Examples: Form inputs, shopping cart, theme settings

🪝 State Hooks

useState provides state management. Each state update triggers a re-render.

Examples: counters, form inputs, toggles, accordions

⚡ Batched Updates

React batches multiple setState calls into one render for performance.

Examples: Multiple state updates in one function

Effects & Lifecycle

useEffect Hook Mastery

🔄 Effect Patterns
// Mount Effect - Runs once useEffect(() => { console.log('Component mounted!'); }, []); // Empty array // Update Effect - Runs on dependency change useEffect(() => { document.title = `Count: ${count}`; }, [count]); // Runs when count changes // Cleanup Effect - Cleanup on unmount useEffect(() => { const timer = setInterval(tick, 1000); return () => clearInterval(timer); // Cleanup }, []);

🎯 Dependency Array

Controls when effect runs: [] = once, [x,y] = when x or y changes, no array = every render

Examples: useEffect(setup, []) for API calls

🧹 Cleanup Function

Return a function to clean up subscriptions, timers, or DOM listeners before next effect.

Examples: clearInterval, removeEventListener, cancel API calls

📡 Data Fetching

Use effects for async operations. Always handle loading and error states.

Examples: fetch API, setTimeout, subscriptions

🔁 Infinite Loop Prevention

Always include dependencies or use useCallback to prevent infinite re-renders.

Examples: Missing dependencies, object function references

Knowledge Check

Quiz Your Understanding

What is JSX?
A syntax extension for JavaScript that allows you to write HTML-like code in React components.
When should you use useState?
For any data that changes over time and affects the UI. Always prefer functional updates for state based on previous state.
What's the purpose of keys in lists?
Keys help React identify which items have changed, are added, or removed, making list updates efficient.
Why is state immutable?
Immutability ensures React can detect changes and efficiently update only what's necessary using reference comparison.
What's the cleanup function in useEffect?
A function returned from useEffect that runs before the next effect execution or component unmounting.
When should you extract a component?
When a piece of UI is reused multiple times, has complex rendering logic, or should manage its own state.

🚀 Pro Developer Tips

  • Pattern Recognition: React is 80% patterns, 20% syntax. Learn the patterns and you can solve any problem.
  • Debugging Strategy: Add console.log in state setters, effects, and render functions to trace execution flow.
  • Performance Mindset: Always think about what re-renders when state changes. Use React DevTools Profiler.
  • Component Naming: Use PascalCase for components, camelCase for variables/properties/functions.
  • Error Boundaries: Wrap components that might throw errors in error boundary components for better UX.
  • Prop Types: Consider using prop-types or TypeScript for large projects to catch prop-passing errors.
